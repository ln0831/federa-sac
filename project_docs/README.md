# Project Docs Hub

This folder is the curated entrypoint for the full workspace.

Use it when you want a cleaner starting point than jumping between `docs/`, `project/`, and `outputs/` manually.

## Reading Order

1. `PROJECT_DOCUMENT.md`
2. `TOP_Q1_SUBMISSION_EXECUTION_PLAN.md`
3. `runtime/README.md`
4. `research/README.md`
5. `results/README.md`
6. `archive/README.md`

## What Lives Here

- `runtime/`: runnable bundle docs, environment notes, runbooks, and command references
- `research/`: paper-writing and research-management document map
- `results/`: experiment evidence map and suite usage guidance
- `archive/`: legacy notes, failed-debug artifacts, and historical traces

## Fast Entry Points

## Working Rule

Treat this folder as the human-facing documentation hub.

The repository root remains the operational source of truth for runnable Python code.
